pinus 后端折腾笔记

发表于 2023-02-24 分类于 后端 ， pinus 阅读次数： Waline： 本文字数： 3.2k 阅读时长 ≈ 3 分钟

pinus 是依托与网易后端框架 pomelo ，采用的是 ts 代码，具体参看 1，我这里主要是折腾利用这个框架写的一个 MMO 游戏。

环境

首先需要有一台 Linux/MacOS 电脑，需要安装对应的工具：

  • nodej
  • mysql

nodejs18

我这里用的是 debian10，使用 apt 指令安装的 npm 跟 nodejs 版本不是最新的，导致出现版本不匹配的问题，因此我这里使用的是如下方法安装 npm/nodejs

删除旧版本的 PPA

sudo add-apt-repository -y -r ppa:chris-lea/node.js
sudo rm -f /etc/apt/sources.list.d/chris-lea-node_js-*.list
sudo rm -f /etc/apt/sources.list.d/chris-lea-node_js-*.list.save

增加 Node 包的 key

sudo apt install curl
KEYRING=/usr/share/keyrings/nodesource.gpg
curl -fsSL https://deb.nodesource.com/gpgkey/nodesource.gpg.key | gpg --dearmor | sudo tee "$KEYRING" >/dev/null

# if using wget
sudo apt install wget
wget --quiet -O - https://deb.nodesource.com/gpgkey/nodesource.gpg.key | gpg --dearmor | sudo tee "$KEYRING" >/dev/nu

安装完毕后，可以查看 key

$ gpg --no-default-keyring --keyring "$KEYRING" --list-keys

/usr/share/keyrings/nodesource.gpg
----------------------------------
pub   rsa4096 2014-06-13 [SC]
      9FD3B784BC1C6FC31A8A0A1C1655A0AB68576280 <- Our signing key has been added.
uid           [ unknown] NodeSource <gpg@nodesource.com>
sub   rsa4096 2014-06-13 [E]

增加 node 源

VERSION=node_18.x
KEYRING=/usr/share/keyrings/nodesource.gpg
DISTRO="$(lsb_release -s -c)"
echo "deb [signed-by=$KEYRING] https://deb.nodesource.com/$VERSION $DISTRO main" | sudo tee /etc/apt/sources.list.d/nodesource.list
echo "deb-src [signed-by=$KEYRING] https://deb.nodesource.com/$VERSION $DISTRO main" | sudo tee -a /etc/apt/sources.list.d/nodesource.list

然后更新 apt 工具

sudo apt update && sudo apt upgrade

最后就是安装 nodejs 了

sudo apt install nodejs

安装完毕，可以查看 node 的版本

$ node -v
v18.0.0

$ npm --version
9.5.1

mysql

mysql 安装就比较简单了，网上有很多资料。

拉取代码

然后就是拉取代码分支：

git clone https://github.com/yangfch3/lordofpomelo

安装依赖

进入根目录，运行 npm-install.sh

cd lordofpomelo
sh npm-install.sh

配置数据

mysql 数据需要先配置好才能使用，因此需要进入数据库进行配置

$ mysql -uroot -p123456
    mysql> create database Pomelo;
    mysql> use Pomelo;
    mysql> source ./game-server/config/schema/Pomelo.sql;

最后需要修改代码中的数据库链接配置：

// lordofpomelo/shared/config/mysql.json
{
    "development": 
    {
        "host": "127.0.0.1",
        "port": "3306",
        "database": "Pomelo",
        "user": "root",
        "password": "123456"
    },
    "production":
    {
        "host" : "127.0.0.1",
        "port" : "3306",
        "database" : "Pomelo",
        "user" : "root",
        "password" : "123456"
    }
}

如果运行后出现如下问题：

{
  code: undefined,
  msg: 'ER_NOT_SUPPORTED_AUTH_MODE: Client does not support authentication 
    protocol requested by server; consider upgrading MySQL client'
}

可能是数据库链接配置有问题，查看数据库地址，用户名以及密码是否正确，如果都正确，下面的方式可能是一种解决方案：

~$: mysql -uroot -p123456
    mysql> ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456';
    mysql> FLUSH PRIVILEGES;
    mysql> exti;

重启服务器，客户端服务器后试试。

这里可以写一个简单的代码测试下：

// testmysql.js
et mysql = require('mysql');

let connection = mysql.createConnection({
    host     : 'localhost',
    user     : 'root',
    password : '********',
    database : 'DB_App_00',
});

connection.connect(function(err) {
    if (err) {
      return console.error('error: ' + err.message);
    }
  
    console.log('Connected to the MySQL server.');
  });

在这个目录 npm install mysql，然后直接 node testmyqsl.js，运行看效果

运行服务器

我这里没能使用文档中的方式运行，我用的是如下指令：

~$: cd game-server
~$: node app

运行客户端服务器

同样运行游戏客户端服务器

~$: cd web-server
~$: node app
Web server has started.
 Please log on http://127.0.0.1:3001/

测试

我这里用的是腾讯云服务器，系统是 Debian10，没有桌面系统，因此无法直接在服务器上运行本机地址测试，因此我在服务器配置里打开了对应的端口 3001 的权限，直接使用对应服务器的 ip + 端口，登陆游戏。

登陆报错

创建角色，登陆都能正常，但是卡在登陆界面，出现如下报错，还在排查:

111 login!
(node:17098) [DEP0106] DeprecationWarning: crypto.createCipher is deprecated.

原因是 gate 端口没开放，需要开放 gate 服务端口。
